# Pathway Documentation

Purpose: route maintainers to the right documentation home for this repository.
Read this first when adding or looking for project documentation.
This page does not define SDK behavior or implementation details.

## Current Documents

- [Documentation Policy](./policy.md): where each kind of document belongs and how documents should be named.

## Document Classes

- `spec/`: normative behavior, contracts, schemas, invariants, and compatibility requirements.
- `runbook/`: executable procedures, validation steps, release checks, and troubleshooting flows.
- `reference/`: descriptive current-state notes about implementation layout or ownership.
- `decisions/`: durable rationale for accepted tradeoffs.

Create a class directory only when adding a real document to it. Do not add empty folders.
